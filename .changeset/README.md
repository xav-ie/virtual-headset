# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish your code. You can
find the full documentation for it [in our repository](https://github.com/changesets/changesets).

We have a quick list of common questions to get you started engaging with this project in
[our documentation](https://github.com/changesets/changesets/blob/main/docs/common-questions.md).

## How releases work here

1. Add a changeset for your change: `just changeset` (or `npm run changeset`), then commit the
   generated `.changeset/*.md`.
2. On merge to `main`, `.github/workflows/version.yml` opens/updates a **"chore: version packages"**
   PR that bumps the version, syncs it into `extension/static/manifest.json`, and writes
   `CHANGELOG.md`.
3. Merging that PR pushes a `v<version>` git tag, which triggers
   [`release.yml`](../.github/workflows/release.yml) to sign the extension (Mozilla, unlisted) and
   publish the signed `.xpi` + `updates.json` to a GitHub Release.
